   N.B, -> A gist on how to read the [CG Bible](https://link.springer.com/book/10.1007/978-3-540-77974-2) to build Maps from scratch(read these chapters in order as mentioned below) -           
          a) [Point Location](https://link.springer.com/chapter/10.1007/978-3-540-77974-2_6)([Point Cloud](https://en.m.wikipedia.org/wiki/Point_cloud))            
          b) [Convex Hull](https://link.springer.com/chapter/10.1007/978-3-540-77974-2_11), [Voronoi Diagrams](https://link.springer.com/chapter/10.1007/978-3-540-77974-2_7) & [Delauny Triangulation](https://link.springer.com/chapter/10.1007/978-3-540-77974-2_9) - lets you build the single layer map           
          c) [Line Segment Intersection](https://link.springer.com/chapter/10.1007/978-3-540-77974-2_2) - overlaying one map layer over the other to show different features of a region(say a buildings map layer over a roads map layer & so on)             
          d) [QuadTrees](https://link.springer.com/chapter/10.1007/978-3-540-77974-2_14) & [More Geometric Structures](https://link.springer.com/chapter/10.1007/978-3-540-77974-2_10) - Storing the maps using computationally efficient Data Structures       
          e) [Orthogonal](https://link.springer.com/chapter/10.1007/978-3-540-77974-2_5) & [Simplex](https://link.springer.com/chapter/10.1007/978-3-540-77974-2_16) Range Searching - finding nearby places(e.g. restaurants or movie theatres) or nearby friends(although this app is more dynamic compared to the nearby places apps since your friends can move)      
          f) [Visibility Graphs](https://link.springer.com/chapter/10.1007/978-3-540-77974-2_15) & [Robot Motion Planning](https://link.springer.com/chapter/10.1007/978-3-540-77974-2_13) - how to move efficiently through the map(basically its all related to graph theory)           
          g) Other Chapters like [Arrangements & Duality](https://link.springer.com/chapter/10.1007/978-3-540-77974-2_8), [Binary Space Partitions](https://link.springer.com/chapter/10.1007/978-3-540-77974-2_12), [Polygon Triangulation](https://link.springer.com/chapter/10.1007/978-3-540-77974-2_3) & [Linear Programming](https://link.springer.com/chapter/10.1007/978-3-540-77974-2_4) are more related to Computer Graphics & CAD/CAM.

       N.B, -> Combinatorial problems can be divided into -        
          a) Whether some permutation/combination of things is possible or not.                     
          b) If possible, what are all the permutations/combinations possible.                
          c) Amongst all the possible permutations/combinations, what's the best(e.g. min or max) or may need to layout the sorted order of permutations/combinations, based on some criteria(& that's what Combinatorial Optimization deals with).
